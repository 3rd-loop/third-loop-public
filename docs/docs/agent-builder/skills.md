# Skills

Skills define specific capabilities for your agents. A skill tells an agent what it's good at and when it should use that expertise. Skills are reusable — you can attach the same skill to multiple agents.

## Creating a Skill

1. Go to the **Create** page and switch to the **Skills** tab.
2. Click the **New** button and select **Skill**.
3. Fill in the following fields:
   - **Name** — A short, descriptive name for the skill.
   - **Purpose** — A brief explanation of what this skill does. This helps the agent understand when to apply it.
   - **Content** — The detailed definition of the skill. This is where you write the prompt or knowledge that powers the skill.
4. Optionally add **trigger conditions** and **tags**.
5. Click **Create**.

## Trigger Conditions

Trigger conditions tell the agent when a skill should activate. You can add multiple conditions to a single skill. Each condition is a plain-text description of a scenario.

For example, a "Code Review" skill might have trigger conditions like:
- "When the user asks for a code review"
- "When the user shares a code snippet and asks for feedback"
- "When the user asks about best practices for a piece of code"

These conditions help the agent decide when to bring this skill into play versus relying on its other skills or general knowledge.

## Managing Skills

Your skills appear in the left sidebar when the Skills tab is selected. You can:

- **Select** a skill to view and edit it.
- **Rename** a skill from the menu icon.
- **Delete** a skill from the menu icon. You will be asked to confirm.
- **Search** for skills using the search bar.

## Tags

Add tags to your skills for organization. Tags help you categorize skills by topic, department, or however makes sense for your use case.
